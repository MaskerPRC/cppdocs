张量基础 ¶

支持 PyTorch 的 ATen 张量库是一个简单的张量库，它将 Torch 中的张量操作直接暴露在 C++17 中。ATen 的 API 是从 PyTorch 使用的相同声明自动生成的，因此这两个 API 将随着时间的推移而相互跟踪。

张量类型是动态解决的，这样 API 就是通用的，不包含模板。也就是说，有一个 Tensor 类型。它可以存储 CPU 或 CUDA 张量，张量可以包含双精度浮点数、浮点数、整数等。这种设计使得编写通用代码变得容易，无需对一切进行模板化。

请参阅 https://pytorch.org/cppdocs/api/namespace_at.html#functions 以获取提供的 API。摘录：

Tensor atan2(const Tensor & other) const;
Tensor & atan2_(const Tensor & other);
Tensor pow(Scalar exponent) const;
Tensor pow(const Tensor & exponent) const;
Tensor & pow_(Scalar exponent);
Tensor & pow_(const Tensor & exponent);
Tensor lerp(const Tensor & end, Scalar weight) const;
Tensor & lerp_(const Tensor & end, Scalar weight);
Tensor histc() const;
Tensor histc(int64_t bins) const;
Tensor histc(int64_t bins, Scalar min) const;
Tensor histc(int64_t bins, Scalar min, Scalar max) const;

同时也提供了原地操作，并且总是以 _ 结尾以表示它们将修改 Tensor。

Tensor 元素的高效访问

当使用 Tensor 广播操作时，动态调用的相对成本非常小。然而，在某些情况下，尤其是在您自己的内核中，需要高效的元素级访问，并且元素级循环中动态调用的成本非常高。ATen 提供了访问器，这些访问器通过单个动态检查来创建，检查 Tensor 是否为所需的类型和维度。然后，访问器公开了一个 API，用于高效访问 Tensor 元素。

访问器是张量的临时视图。它们仅在其所查看的张量生命周期内有效，因此应仅在函数中本地使用，例如迭代器。

注意，访问器与内核函数中的 CUDA 张量不兼容。相反，您将不得不使用打包访问器，它具有相同的行为，但会复制张量元数据而不是指向它。

因此建议使用访问器处理 CPU 张量，使用打包访问器处理 CUDA 张量。

CPU 访问器 ¶

torch::Tensor foo = torch::rand({12, 12});

// assert foo is 2-dimensional and holds floats.
auto foo_a = foo.accessor<float,2>();
float trace = 0;

for(int i = 0; i < foo_a.size(0); i++) {
  // use the accessor foo_a to get tensor data.
  trace += foo_a[i][i];
}

CUDA 访问器 ¶

__global__ void packed_accessor_kernel(
    torch::PackedTensorAccessor64<float, 2> foo,
    float* trace) {
  int i = threadIdx.x;
  gpuAtomicAdd(trace, foo[i][i]);
}

torch::Tensor foo = torch::rand({12, 12});

// assert foo is 2-dimensional and holds floats.
auto foo_a = foo.packed_accessor64<float,2>();
float trace = 0;

packed_accessor_kernel<<<1, 12>>>(foo_a, &trace);

除了 PackedTensorAccessor64 和 packed_accessor64 之外，还有相应的 PackedTensorAccessor32 和 packed_accessor32 ，它们使用 32 位整数进行索引。这在 CUDA 上可能会快得多，但可能会导致索引计算溢出。

注意，模板可以包含其他参数，例如指针限制和索引的整数类型。请参阅文档以获取访问器和打包访问器的详细模板描述。

使用外部创建的数据 ¶

如果你已经在内存中分配了你的张量数据（CPU 或 CUDA），你可以在 ATen 中将该内存视为 Tensor ：

float data[] = { 1, 2, 3,
                 4, 5, 6 };
torch::Tensor f = torch::from_blob(data, {2, 3});

这些张量不能调整大小，因为 ATen 不拥有内存，但其他方面与正常张量表现一致。

标量和零维张量

除了 Tensor 对象外，ATen 还包括 Scalar ，它们代表一个单独的数字。与张量一样，标量是动态类型的，可以持有 ATen 的任何一种数字类型。标量可以从 C++数字类型隐式构造。标量是必需的，因为一些函数如 addmm 需要数字和张量，并期望这些数字与张量具有相同的动态类型。它们还用于 API 中，以指示函数将始终返回标量值，如 sum 。

namespace torch {
Tensor addmm(Scalar beta, const Tensor & self,
             Scalar alpha, const Tensor & mat1,
             const Tensor & mat2);
Scalar sum(const Tensor & self);
} // namespace torch

// Usage.
torch::Tensor a = ...
torch::Tensor b = ...
torch::Tensor c = ...
torch::Tensor r = torch::addmm(1.0, a, .5, b, c);

除了 Scalar s，ATen 还允许 Tensor 对象是零维的。这些张量包含单个值，并且可以引用更大的 Tensor 中的一个单个元素。它们可以在需要 Tensor 的任何地方使用。它们通常通过 select 等运算符创建，这些运算符会减少张量的维度。

torch::Tensor two = torch::rand({10, 20});
two[1][2] = 4;
// ^^^^^^ <- zero-dimensional Tensor
