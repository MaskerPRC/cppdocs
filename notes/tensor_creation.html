张量创建 API ¶

本笔记描述了如何在 PyTorch C++ API 中创建张量。它强调了可用的工厂函数，这些函数根据某些算法填充新的张量，并列出了可用于配置新张量形状、数据类型、设备和其他属性的选项。

工厂函数 ¶

工厂函数是一种生成新张量的函数。PyTorch（包括 Python 和 C++）中提供了许多工厂函数，它们在初始化新张量并返回之前的方式上有所不同。所有工厂函数都遵循以下一般的“模式”：

torch::<function-name>(<function-specific-options>, <sizes>, <tensor-options>)

让我们分解这个“模式”的各个部分：

  1. <function-name> 是您想要调用的函数名称，

  2. <functions-specific-options> 是特定工厂函数接受的任何必需或可选参数，

  3. <sizes> 是一个类型为 IntArrayRef 的对象，并指定了结果张量的形状，

  4. <tensor-options> 是 TensorOptions 的一个实例，并配置了结果张量的数据类型、设备、布局和其他属性。

选择工厂函数

以下是在撰写本文时（超链接指向相应的 Python 函数，因为它们通常有更详细的文档——选项相同）可用的工厂函数：

  • arange: 返回一个包含整数序列的张量，

  • empty: 返回一个未初始化值的张量，

  • eye: 返回一个单位矩阵，

  • full: 返回一个填充单一值的张量，

  • linspace：返回一个在某个区间内线性间隔的张量，

  • logspace：返回一个在某个区间内对数间隔的张量，

  • ones：返回一个填充所有 1 的张量，

  • rand：返回一个从均匀分布中抽取值的张量，

  • randint：返回一个从区间中随机抽取整数的张量，

  • randn：返回一个填充有从单位正态分布中抽取的值的张量，

  • randperm：返回一个填充有某些区间内随机排列的整数的张量，

  • zeros：返回一个填充全零的张量。

指定大小

一些函数由于其填充张量的方式，本质上不需要特定的参数，可以使用大小来调用。例如，以下行创建了一个包含 5 个元素的向量，初始值都设置为 1：

torch::Tensor tensor = torch::ones(5);

如果我们想创建一个 3 x 5 矩阵，或者一个 2 x 3 x 4 张量呢？一般来说， IntArrayRef （工厂函数的大小参数类型）是通过指定每个维度的尺寸来构造的。例如， {2, 3} 表示一个具有两行三列的张量（在这种情况下是矩阵）， {3, 4, 5} 表示一个三维张量， {2} 表示一个包含两个组件的一维张量。在一维情况下，您可以省略花括号，只需传递单个整数即可，就像我们上面做的那样。请注意，波浪形花括号只是构造 IntArrayRef 的一种方式。您还可以传递 std::vector<int64_t> 和其他几种类型。无论如何，这意味着我们可以通过以下方式构造一个填充有单位正态分布值的三个维度的张量：

torch::Tensor tensor = torch::randn({3, 4, 5});
assert(tensor.sizes() == std::vector<int64_t>{3, 4, 5});

tensor.sizes() 返回一个 IntArrayRef ，可以与一个 std::vector<int64_t> 进行比较，我们可以看到它包含了我们传递给张量的尺寸。您还可以写入 tensor.size(i) 来访问单个维度，这相当于但优于 tensor.sizes()[i] 。

传递函数特定参数

ones 和 randn 都不接受任何额外的参数来改变它们的行为。一个需要进一步配置的函数是 randint ，它接受生成整数的上界，以及一个可选的下界，默认为 0。这里我们创建一个 5 x 5 的平方矩阵，矩阵中的整数介于 0 到 10 之间：

torch::Tensor tensor = torch::randint(/*high=*/10, {5, 5});

然后，我们将下界提高到 3：

torch::Tensor tensor = torch::randint(/*low=*/3, /*high=*/10, {5, 5});

内联注释 /*low=*/ 和 /*high=*/ 当然不是必需的，但就像 Python 中的关键字参数一样，有助于提高可读性。

提示

主要的收获是大小总是跟随特定函数的参数。

注意

有时候一个函数根本不需要大小参数。例如， arange 返回的张量的大小完全由其特定函数的参数指定——整数范围的上下限。在这种情况下，函数不需要 size 参数。

配置张量的属性

前一节讨论了特定函数的参数。特定函数的参数只能改变填充张量的值，有时还可以改变张量的大小。它们永远不会改变创建的张量的数据类型（例如 float32 或 int64 ）或它是否位于 CPU 或 GPU 内存中。这些属性的指定留给每个工厂函数的最后一个参数：一个下面将要讨论的 TensorOptions 对象。

TensorOptions 是一个封装 Tensor 构建轴的类。构建轴是指 Tensor 在构建之前可以配置（有时也可以在之后更改）的特定属性。这些构建轴包括：

  • dtype （以前称为“标量类型”），它控制存储在张量中的元素的数据类型，

  • layout ，它可以是带状（密集）或稀疏的，

  • device 代表存储张量的计算设备（如 CPU 或 CUDA GPU），

  • requires_grad 布尔值，用于启用或禁用张量的梯度记录，

如果你熟悉 Python 中的 PyTorch，这些轴将非常熟悉。目前这些轴的允许值有：

  • 对于 dtype : kUInt8 ， kInt8 ， kInt16 ， kInt32 ， kInt64 ， kFloat32 和 kFloat64 ，

  • 对于 layout : kStrided 和 kSparse ，

  • 对于 device : 要么 kCPU ，要么 kCUDA （可接受可选的设备索引），

  • 对于 requires_grad : 要么 true 要么 false 。

提示

存在类似于 Rust 风格的简写形式用于 dtypes，如 kF32 替代 kFloat32 。完整列表请见此处。

TensorOptions 的一个实例为每个这些轴存储一个具体的值。以下是一个创建表示 64 位浮点、需要梯度且位于 CUDA 设备 1 上的 TensorOptions 对象的示例：

auto options =
  torch::TensorOptions()
    .dtype(torch::kFloat32)
    .layout(torch::kStrided)
    .device(torch::kCUDA, 1)
    .requires_grad(true);

注意我们如何使用 TensorOptions 的“builder”风格方法逐步构建对象。如果我们把这个对象作为工厂函数的最后一个参数传递，新创建的张量将具有以下属性：

torch::Tensor tensor = torch::full({3, 4}, /*value=*/123, options);

assert(tensor.dtype() == torch::kFloat32);
assert(tensor.layout() == torch::kStrided);
assert(tensor.device().type() == torch::kCUDA); // or device().is_cuda()
assert(tensor.device().index() == 1);
assert(tensor.requires_grad());

现在，你可能想知道：我是否真的需要为每个新创建的张量指定每个轴？幸运的是，答案是否定的，因为每个轴都有一个默认值。这些默认值是：

  • kFloat32 用于数据类型，

  • 用于布局的

  • 用于设备的

  • 用于 requires_grad 的

这意味着在构建一个 TensorOptions 对象时省略的任何轴都将采用其默认值。例如，这是我们之前的 TensorOptions 对象，但 dtype 和 layout 已默认：

auto options = torch::TensorOptions().device(torch::kCUDA, 1).requires_grad(true);

事实上，我们甚至可以省略所有轴，得到一个完全默认的 TensorOptions 对象：

auto options = torch::TensorOptions(); // or `torch::TensorOptions options;`

这个结果的美好之处在于，我们刚才谈了这么多的 TensorOptions 对象可以完全从任何张量工厂调用中省略：

// A 32-bit float, strided, CPU tensor that does not require a gradient.
torch::Tensor tensor = torch::randn({3, 4});
torch::Tensor range = torch::arange(5, 10);

但甜头更大：到目前为止，在所提供的 API 中，你可能已经注意到初始的 torch::TensorOptions() 相当长。好消息是，对于每个构造轴（数据类型、布局、设备以及 requires_grad ），在 torch:: 命名空间中都有一个免费函数，你可以为该轴传递一个值。每个函数然后返回一个预配置了该轴的 TensorOptions 对象，但允许通过上述构建器样式方法进行进一步修改。例如，

torch::ones(10, torch::TensorOptions().dtype(torch::kFloat32))

等价于

torch::ones(10, torch::dtype(torch::kFloat32))

并且进一步地，我们可以直接写

torch::ones(10, torch::TensorOptions().dtype(torch::kFloat32).layout(torch::kStrided))

我们可以简单地写

torch::ones(10, torch::dtype(torch::kFloat32).layout(torch::kStrided))

这样可以节省我们很多打字。这意味着在实践中，你几乎永远不需要写出 torch::TensorOptions 。相反，使用 torch::dtype() 、 torch::device() 、 torch::layout() 和 torch::requires_grad() 函数。

最后，一个便利之处在于 TensorOptions 可以隐式地从单个值构建。这意味着每当一个函数有一个 TensorOptions 类型的参数，就像所有工厂函数一样，我们可以直接传递一个值，如 torch::kFloat32 或 torch::kStrided ，来代替整个对象。因此，当我们只想更改与默认值相比的单个轴时，我们只需传递那个值。因此，原本

torch::ones(10, torch::TensorOptions().dtype(torch::kFloat32))

变成了

torch::ones(10, torch::dtype(torch::kFloat32))

并且可以最终缩短为

torch::ones(10, torch::kFloat32)

当然，使用这种简短语法无法进一步修改 TensorOptions 实例的更多属性，但如果我们只需要更改一个属性，这相当实用。

总结来说，我们现在可以比较如何使用 TensorOptions 默认值以及使用自由函数创建 TensorOptions 的简写 API，在 C++中创建张量与在 Python 中具有相同的便利性。比较以下 Python 中的调用：

torch.randn(3, 4, dtype=torch.float32, device=torch.device('cuda', 1), requires_grad=True)

C++中的等效调用为：

torch::randn({3, 4}, torch::dtype(torch::kFloat32).device(torch::kCUDA, 1).requires_grad(true))

非常接近！

转换 §

就像我们可以使用 TensorOptions 来配置如何创建新的张量一样，我们也可以使用 TensorOptions 将张量从一组属性转换为另一组属性。这种转换通常创建一个新的张量，并且不会原地发生。例如，如果我们有一个使用 source_tensor 创建的张量

torch::Tensor source_tensor = torch::randn({2, 3}, torch::kInt64);

我们可以将它从 int64 转换为 float32 :

torch::Tensor float_tensor = source_tensor.to(torch::kFloat32);

注意

转换的结果， float_tensor ，是一个指向新内存的新张量，与源 source_tensor 无关。

然后，我们可以将其从 CPU 内存移动到 GPU 内存：

torch::Tensor gpu_tensor = float_tensor.to(torch::kCUDA);

如果你有多个 CUDA 设备可用，上述代码将把张量复制到默认的 CUDA 设备，你可以通过 torch::DeviceGuard 来配置它。如果没有 DeviceGuard ，这将默认为 GPU 1。如果你想指定不同的 GPU 索引，你可以将其传递给 Device 构造函数：

torch::Tensor gpu_two_tensor = float_tensor.to(torch::Device(torch::kCUDA, 1));

在 CPU 到 GPU 复制和反向操作的情况下，我们还可以通过将 /*non_blocking=*/false 作为 to() 的最后一个参数传递来配置内存复制为异步：

torch::Tensor async_cpu_tensor = gpu_tensor.to(torch::kCPU, /*non_blocking=*/true);

结论 ¶

希望这篇笔记能让你对如何使用 PyTorch C++ API 以惯用方式创建和转换张量有一个良好的理解。如果你有任何进一步的问题或建议，请使用我们的论坛或 GitHub 问题来联系我们。
