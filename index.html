PyTorch C++ API 简介

这些页面提供了 PyTorch C++ API 公共部分的文档。此 API 大致可以分为五个部分：

  • ATen：所有其他功能构建在之上的基础张量和数学运算库。

  • Autograd：增强 ATen 的自动微分功能。

  • C++前端：用于训练和评估机器学习模型的高级结构。

  • TorchScript：TorchScript 即时编译器和解释器的接口。

  • C++扩展：通过自定义 C++和 CUDA 例程扩展 Python API 的一种方式。

这些构建块结合在一起，形成了一个针对张量计算和动态神经网络的研究和生产就绪的 C++库，该库对 GPU 加速和快速 CPU 性能有很强的重视。目前该库已在 Facebook 的研究和生产中使用；我们期待欢迎更多 PyTorch C++ API 的用户。

警告

目前，C++ API 应被视为“beta”稳定性；我们可能会为了改进 API 或提供 Python 接口到 PyTorch（这是我们最稳定且支持最好的接口）而对后端进行重大破坏性更改。

ATen¶

ATen 基本上是一个张量库，PyTorch 中几乎所有其他 Python 和 C++ 接口都是建立在它之上的。它提供了一个核心 Tensor 类，许多数百个操作都定义在这个类上。其中大部分操作都有 CPU 和 GPU 实现， Tensor 类将根据其类型动态调度到这些实现。使用 ATen 的小例子可能如下所示：

#include <ATen/ATen.h>

at::Tensor a = at::ones({2, 2}, at::kInt);
at::Tensor b = at::randn({2, 2});
auto c = a + b.to(at::kInt);

ATen 中的此类以及所有其他符号都位于此处文档的 at:: 命名空间中。

自动微分 ¶

我们所说的 autograd 是 PyTorch 的 C++ API 中增强 ATen Tensor 类自动微分功能的部分。autograd 系统记录张量上的操作以形成 autograd 图。在此图中对叶变量调用 backwards() 将执行通过跨越 autograd 图的函数和张量网络的反向模式微分，最终得到梯度。以下示例展示了此接口的滋味：

#include <torch/csrc/autograd/variable.h>
#include <torch/csrc/autograd/function.h>

torch::Tensor a = torch::ones({2, 2}, torch::requires_grad());
torch::Tensor b = torch::randn({2, 2});
auto c = a + b;
c.backward(); // a.grad() will now hold the gradient of c w.r.t. a.

ATen 中的 at::Tensor 类默认情况下不可微分。要添加 autograd API 提供的张量可微性，您必须使用 torch::命名空间中的张量工厂函数，而不是 at::命名空间。例如，使用 at::ones 创建的张量将不可微分，而使用 torch::ones 创建的张量将是可微分的。

C++前端

PyTorch 的 C++前端提供了一个高级、纯 C++的建模接口，用于神经网络和通用机器学习（ML）研究和生产用例，在设计和功能上很大程度上遵循 Python API。C++前端包括以下内容：

  • 通过分层模块系统（如 torch.nn.Module ）定义机器学习模型的接口；

  • 一个“标准库”，包含用于最常见建模目的的预存在模块（例如卷积、RNN、批量归一化等）；

  • 一个优化 API，包括流行的优化器实现，如 SGD、Adam、RMSprop 等；

  • 表示数据集和数据管道的方法，包括在多个 CPU 核心上并行加载数据的功能；

  • 存储和加载训练会话检查点的序列化格式（如 torch.utils.data.DataLoader ）；

  • 自动将模型并行化到多个 GPU 上（如 torch.nn.parallel.DataParallel ）；

  • 支持代码，可轻松使用 pybind11 将 C++模型绑定到 Python；

  • 火炬脚本 JIT 编译器的入口点；

  • 有助于与 ATen 和 Autograd API 交互的实用工具。

请参阅此文档以获取 C++ 前端的更详细描述。与 C++ 前端相关的 torch:: 命名空间的相关部分包括 torch::nn、torch::optim、torch::data、torch::serialize、torch::jit 和 torch::python。C++ 前端的示例可以在本存储库中找到，该存储库正在持续和积极地进行扩展。

备注

除非您有特定的理由要仅限于 ATen 或 Autograd API，否则建议使用 C++ 前端作为 PyTorch C++ 生态系统的入口点。虽然它仍在收集用户反馈（来自您！）的测试版中，但它提供了比 ATen 和 Autograd API 更多的功能以及更好的稳定性保证。

TorchScript

TorchScript 是 PyTorch 模型的表示，可以被 TorchScript 编译器理解、编译和序列化。从根本上讲，TorchScript 是一种编程语言。它是使用 PyTorch API 的 Python 子集。TorchScript 的 C++ 接口包括三个主要功能：

  • 一种加载和执行在 Python 中定义的序列化 TorchScript 模型的机制；

  • 一个用于定义自定义操作符的 API，以扩展 TorchScript 标准库的操作；

  • TorchScript 程序的即时编译从 C++。

如果您尽可能希望在 Python 中定义模型，但随后将它们导出到 C++用于生产环境和无 Python 推理，那么第一个机制可能对您非常有用。您可以点击此链接了解更多信息。第二个 API 关注的是您希望扩展 TorchScript 以自定义操作符的场景，这些操作符可以像从 C++中序列化和调用一样进行序列化和调用。最后，torch::jit::compile 函数可以用于从 C++直接访问 TorchScript 编译器。

C++扩展

C++扩展提供了一种简单而强大的方式来访问上述所有接口，用于扩展 PyTorch 的常规 Python 用例。C++扩展最常用于在 C++或 CUDA 中实现自定义操作符，以加速 vanilla PyTorch 设置中的研究。C++扩展 API 不会向 PyTorch C++ API 添加任何新功能。相反，它提供了与 Python setuptools 的集成以及 JIT 编译机制，允许从 Python 访问 ATen、autograd 和其他 C++ API。要了解更多关于 C++扩展 API 的信息，请参阅这篇教程。
