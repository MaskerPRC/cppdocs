C++ 前端 ¶

PyTorch C++前端是一个用于 CPU 和 GPU 张量计算的 C++17 库，具有自动微分和高级构建块，适用于最先进的机器学习应用。

描述 ¶

可以将 PyTorch C++前端视为 PyTorch Python 前端的 C++版本，提供自动微分和各种高级抽象，用于机器学习和神经网络。具体来说，它包括以下组件：

组件

描述

torch::Tensor

自动微分、高效支持 CPU 和 GPU 的张量

torch::nn

用于神经网络建模的可组合模块集合

torch::optim

用于训练模型的优化算法，如 SGD、Adam 或 RMSprop

torch::data

数据集、数据管道和多线程、异步数据加载器

torch::serialize

存储和加载模型检查点的序列化 API

torch::python

将您的 C++模型绑定到 Python 的粘合剂

torch::jit

对 TorchScript JIT 编译器的纯 C++访问

端到端示例 ¶

这是一个定义和训练简单神经网络在 MNIST 数据集上的简单、端到端示例：

#include <torch/torch.h>

// Define a new Module.
struct Net : torch::nn::Module {
  Net() {
    // Construct and register two Linear submodules.
    fc1 = register_module("fc1", torch::nn::Linear(784, 64));
    fc2 = register_module("fc2", torch::nn::Linear(64, 32));
    fc3 = register_module("fc3", torch::nn::Linear(32, 10));
  }

  // Implement the Net's algorithm.
  torch::Tensor forward(torch::Tensor x) {
    // Use one of many tensor manipulation functions.
    x = torch::relu(fc1->forward(x.reshape({x.size(0), 784})));
    x = torch::dropout(x, /*p=*/0.5, /*train=*/is_training());
    x = torch::relu(fc2->forward(x));
    x = torch::log_softmax(fc3->forward(x), /*dim=*/1);
    return x;
  }

  // Use one of many "standard library" modules.
  torch::nn::Linear fc1{nullptr}, fc2{nullptr}, fc3{nullptr};
};

int main() {
  // Create a new Net.
  auto net = std::make_shared<Net>();

  // Create a multi-threaded data loader for the MNIST dataset.
  auto data_loader = torch::data::make_data_loader(
      torch::data::datasets::MNIST("./data").map(
          torch::data::transforms::Stack<>()),
      /*batch_size=*/64);

  // Instantiate an SGD optimization algorithm to update our Net's parameters.
  torch::optim::SGD optimizer(net->parameters(), /*lr=*/0.01);

  for (size_t epoch = 1; epoch <= 10; ++epoch) {
    size_t batch_index = 0;
    // Iterate the data loader to yield batches from the dataset.
    for (auto& batch : *data_loader) {
      // Reset gradients.
      optimizer.zero_grad();
      // Execute the model on the input data.
      torch::Tensor prediction = net->forward(batch.data);
      // Compute a loss value to judge the prediction of our model.
      torch::Tensor loss = torch::nll_loss(prediction, batch.target);
      // Compute gradients of the loss w.r.t. the parameters of our model.
      loss.backward();
      // Update the parameters based on the calculated gradients.
      optimizer.step();
      // Output the loss and checkpoint every 100 batches.
      if (++batch_index % 100 == 0) {
        std::cout << "Epoch: " << epoch << " | Batch: " << batch_index
                  << " | Loss: " << loss.item<float>() << std::endl;
        // Serialize your model periodically as a checkpoint.
        torch::save(net, "net.pt");
      }
    }
  }
}

哲学 ¶

PyTorch 的 C++前端设计理念是 Python 前端非常出色，应在可能的情况下使用；但在某些设置中，性能和可移植性要求使得使用 Python 解释器不可行。例如，Python 在低延迟、高性能或多线程环境中（如视频游戏或生产服务器）不是一个好的选择。C++前端的目标是解决这些用例，同时不牺牲 Python 前端的用户体验。

因此，C++前端在设计、命名、约定和功能上紧密模仿 Python 前端：

  • 在设计、命名、约定和功能上与 Python 前端保持紧密一致。虽然两个前端之间可能存在一些差异（例如，我们在 Python 前端中删除了已弃用的功能或修复了“瑕疵”），但我们保证将 Python 模型移植到 C++的工作将仅限于翻译语言特性，而不是修改功能或行为。

  • 优先考虑灵活性和用户友好性，而不是微优化。在 C++中，你通常可以得到最优的代码，但代价是极端不友好的用户体验。灵活性和动态性是 PyTorch 的核心，C++前端旨在保持这种体验，在某些情况下，为了保持 API 简单易懂，甚至牺牲性能（或“隐藏”性能控制按钮）。我们希望那些不以编写 C++为生的研究人员能够使用我们的 API。

警告：Python 并不一定比 C++ 慢！Python 前端几乎调用 C++ 来执行任何计算密集型操作（尤其是任何类型的数值运算），这些操作将占用程序中大部分时间。如果您更喜欢编写 Python，并且能够承担编写 Python 的成本，我们建议使用 PyTorch 的 Python 接口。然而，如果您更喜欢编写 C++，或者需要编写 C++（因为多线程、延迟或部署需求），PyTorch 的 C++ 前端提供了一个与 Python 对应物大致相同方便、灵活、友好和直观的 API。这两个前端服务于不同的用例，协同工作，并且没有一个旨在无条件地取代另一个。
