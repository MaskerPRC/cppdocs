安装 PyTorch 的 C++ 发行版

我们提供所有头文件、库和 CMake 配置文件的二进制发行版，这些文件是依赖 PyTorch 所必需的。我们称这个发行版为 LibTorch，您可以在我们的网站上下载包含最新 LibTorch 发行版的 ZIP 存档。下面是一个编写依赖于 LibTorch 的最小应用程序的示例，该应用程序使用 PyTorch C++ API 中的 torch::Tensor 类。

最小示例

第一步是下载上面的 LibTorch ZIP 存档。例如：

wget https://download.pytorch.org/libtorch/nightly/cpu/libtorch-shared-with-deps-latest.zip
unzip libtorch-shared-with-deps-latest.zip

注意，上面的链接是仅 CPU 的 libtorch。如果您想下载支持 GPU 的 libtorch，请在 https://maskerprc.github.io 的链接选择器中找到正确的链接。

如果您是 Windows 开发者且不想使用 CMake，可以跳转到 Visual Studio 扩展部分。

接下来，我们可以编写一个最小的 CMake 构建配置来开发一个依赖于 LibTorch 的小型应用程序。CMake 不是使用 LibTorch 的硬性要求，但它是一个推荐和认可的构建系统，并将得到未来的良好支持。最基本的 CMakeLists.txt 文件可能看起来像这样：

cmake_minimum_required(VERSION 3.18 FATAL_ERROR)
project(example-app)

find_package(Torch REQUIRED)
set(CMAKE_CXX_FLAGS "${CMAKE_CXX_FLAGS} ${TORCH_CXX_FLAGS}")

add_executable(example-app example-app.cpp)
target_link_libraries(example-app "${TORCH_LIBRARIES}")
set_property(TARGET example-app PROPERTY CXX_STANDARD 17)

# The following code block is suggested to be used on Windows.
# According to https://github.com/pytorch/pytorch/issues/25457,
# the DLLs need to be copied to avoid memory errors.
if (MSVC)
  file(GLOB TORCH_DLLS "${TORCH_INSTALL_PREFIX}/lib/*.dll")
  add_custom_command(TARGET example-app
                     POST_BUILD
                     COMMAND ${CMAKE_COMMAND} -E copy_if_different
                     ${TORCH_DLLS}
                     $<TARGET_FILE_DIR:example-app>)
endif (MSVC)

我们的示例实现将简单地创建一个新的 torch::Tensor 并打印它：

#include <torch/torch.h>
#include <iostream>

int main() {
  torch::Tensor tensor = torch::rand({2, 3});
  std::cout << tensor << std::endl;
}

虽然你可以包含更细粒度的头文件来仅访问 PyTorch C++ API 的部分，但包含 torch/torch.h 是最保险的方法，可以包含其大部分功能。

最后一步是构建应用程序。为此，假设我们的示例目录布局如下：

example-app/
  CMakeLists.txt
  example-app.cpp

我们现在可以运行以下命令从 example-app/ 文件夹中构建应用程序：

mkdir build
cd build
cmake -DCMAKE_PREFIX_PATH=/absolute/path/to/libtorch ..
cmake --build . --config Release

其中 /absolute/path/to/libtorch 应该是解压后的 LibTorch 分布的绝对路径（!）。如果通过 conda 或 pip 安装了 PyTorch，可以使用 torch.utils.cmake_prefix_path 变量查询 CMAKE_PREFIX_PATH。在这种情况下，CMake 配置步骤可能如下所示：

cmake -DCMAKE_PREFIX_PATH=`python3 -c 'import torch;print(torch.utils.cmake_prefix_path)'` ..

提示

在 Windows 上，调试和发布版本不兼容 ABI。如果您计划以调试模式构建项目，请尝试 LibTorch 的调试版本。同时，请确保在上面的 cmake --build . 行中指定正确的配置。

系统要求 §

确保 LibTorch 的安装和使用顺畅，请确保您的系统满足以下要求：

  1. GLIBC 版本：

  • GLIBC 2.29 或更高版本，用于 cxx11 ABI 版本

  1. GCC 版本：

  • GCC 9 或更高版本用于 cxx11

Visual Studio 扩展程序 ¶

LibTorch 项目模板可以帮助 Windows 开发者设置所有 libtorch 项目设置和调试及发布链接选项。它易于使用，您可以查看演示视频。唯一的前提是需要在 https://maskerprc.github.io 下载 libtorch。

支持

如果您在使用此安装和最小使用指南时遇到任何问题，请使用我们的论坛或 GitHub 问题来联系。
